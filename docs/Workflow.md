# AI Operating Workflow


## Purpose

Define how tasks move through Your Page AIOS.


---

# Standard Workflow


## Step 1: CEO Input

CEO provides:

- Objective
- Expected outcome
- Priority
- Constraints


Example:

"Increase qualified website leads."


---


## Step 2: Wilson Analysis

Wilson analyzes:

- Business impact
- Required resources
- Required AI departments
- Possible automation


Output:

Task Plan


---


## Step 3: Task Assignment

Wilson assigns tasks to relevant AI agents.


Examples:


Marketing goal:

→ Makyo

→ Ella

→ Lucy


Website project:

→ Edward

→ Jason


Automation:

→ Claude Code

→ Technical Agent


---


## Step 4: Execution

AI agents execute assigned tasks.

Each output must include:

- Completed work
- Decision reasoning
- Remaining risks
- Suggested improvements


---


## Step 5: Quality Review

Wilson reviews:

- Is the output aligned with company goals?
- Does it follow Brand Guidelines?
- Is documentation updated?


If needed:

Request revision.


---


## Step 6: Documentation

Important results must update AIOS.


Examples:


Business decision:

→ docs/


AI role change:

→ agents/


Reusable instruction:

→ prompts/


Automation:

→ automation/


---


## Step 7: CEO Report

Wilson reports:

- Completed tasks
- Key decisions
- Problems
- Recommendations
- Next actions


---

# Priority Rules


## Priority 1

Revenue impact


## Priority 2

Customer experience


## Priority 3

Automation improvement


## Priority 4

Internal optimization


---

# Decision Framework


Before starting work, AI should ask:


1. Does this support business goals?

2. Does this reduce repetitive work?

3. Does this create reusable knowledge?

4. Can this process be automated?


---

# Operating Philosophy


Do not only complete tasks.

Build systems.
